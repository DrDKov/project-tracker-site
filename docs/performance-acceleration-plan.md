# План ускорения React-ветки workspace

Цель: вернуть ощущение мгновенной работы приложения. Создание обычной задачи, переключение режимов задач и точечные realtime-обновления не должны запускать полную перезагрузку workspace.

## Целевые бюджеты

- Создание обычной задачи: визуальный отклик до 100-200 мс, сетевой ответ может завершаться позже.
- Переключение режимов задач: до 100-160 мс на текущем объеме данных.
- Построение task board model: до 120 мс.
- Realtime update: точечный patch без polling, reload и bootstrap invalidate.
- Раздел «Материалы»: не должен перезагружаться из-за событий задач.

## Этап 1. Измерение без изменения поведения

Статус: начат.

Задачи этапа:

1. Зафиксировать план ускорения в репозитории.
2. Добавить диагностические performance helpers без изменения пользовательского поведения.
3. Замерить тяжелые места:
   - `tasks:model` — построение view model раздела задач;
   - `tasks:mode-switch` — переключение режимов «по статусам / по исполнителям / по неделе»;
   - `tasks:save-db-task` — сохранение строки задачи в БД;
   - `tasks:save-db-assignees` — обновление исполнителей;
   - `tasks:save-local-patch` — локальное обновление store и рендерных триггеров;
   - `tasks:save-total` — полное время сохранения обычной задачи.
4. После деплоя смотреть диагностику в консоли:
   - `window.__workspacePerfAudit`
   - `window.__workspacePerfSummary`

Критерий завершения этапа: есть реальные цифры, показывающие, где именно задержка: сеть, локальная модель, render/commit или лишний reload.

## Этап 2. Убрать скрытые full reload/invalidation из task-actions

Нельзя, чтобы обычные действия задач вызывали полный bootstrap workspace.

Проверить и устранить:

- создание задачи;
- редактирование задачи;
- назначение исполнителей;
- drag-drop;
- звездочка;
- завершение задачи;
- подзадачи;
- комментарии.

Исключение: массовое создание повторяющихся задач может временно оставаться с reload, пока сервис не возвращает полные строки созданных задач.

## Этап 3. Нормализовать данные раздела задач

Создать индексы вместо повторных `.filter()` по массивам:

- `tasksById`;
- `activeTaskIds`;
- `subtasksByTaskId`;
- `assigneesByTaskId`;
- `commentsCountByTaskId`;
- `projectById`;
- `userById`.

Цель: карточка задачи не должна самостоятельно фильтровать все подзадачи, комментарии и исполнителей.

## Этап 4. Memoized selectors для режимов задач

Сделать дешевые селекторы:

- `tasksGroupedByStatus`;
- `tasksGroupedByAssignee`;
- `tasksGroupedByWeek`.

Пересчет только при изменении входных данных: `tasks`, `filters`, `weekStart`, `showCompleted`.

## Этап 5. Мемоизация и облегчение TaskCard

- `TaskCard` должен быть `React.memo`.
- В карточку передавать готовую view model или готовые индексы.
- Не пересоздавать тяжелые функции на каждую карточку.
- Ограничить количество видимых подзадач на карточке или добавить collapsed view.

## Этап 6. Optimistic create/update/delete

Обычная задача:

1. Создать локальную временную задачу.
2. Закрыть модалку сразу.
3. Показать карточку в нужной колонке.
4. Сохранить в БД в фоне.
5. Заменить temporary id на реальный id.
6. При ошибке откатить или показать ошибочное состояние.

## Этап 7. Realtime patch-only

Realtime не должен запускать reload.

События:

- `INSERT tasks` — добавить одну задачу;
- `UPDATE tasks` — обновить одну задачу;
- `DELETE` или `deleted_at` — убрать одну задачу и связанные элементы;
- `INSERT task_subtasks` — добавить одну подзадачу;
- `UPDATE task_subtasks` — обновить одну подзадачу;
- `INSERT task_comments` — добавить один комментарий;
- `INSERT task_assignees` — добавить назначение.

## Этап 8. Настоящие уведомления

Оптимальная модель: отдельная таблица `notifications`, которую заполняет серверная функция/trigger при назначении задачи или mention. Клиент подписывается только на уведомления текущего пользователя.

Это уберет необходимость пересчитывать уведомления из `task_comments` и `task_assignees` на клиенте.

## Правило дальнейших изменений

Перед push в `react`:

```bash
npm run check
npm run build
```

Без зеленой локальной проверки не отправлять изменения в рабочую ветку.
